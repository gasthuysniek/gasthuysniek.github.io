SAP Analytics Cloud

In SAP Analytics CLoud a user can execute functionalities in a Story, directly in the application or with a chatbot

Story

Graphical representations

A user can request the desired graphics for his data in the story. To enhance the user experience, the various graphics have been divided into the following categories.

VDT
This graphic shows how the hierarchy between the different accounts is structured.

Table
This graphical representation allows a user to schedule FTE values.

Treshholds
Tresholds can be set in the planning model. These indicate intervals between which values can lie. The current intervals are set that when an account goes below 0, one enters the interval warning of the tresholds. These tresholds are displayed using bullet charts.

GeoMap
A planning model can be built in two ways. One is to start directly from a flat file (Excel) and extract the dimensions in the model directly from there. This way of working makes it possible to create the GeoMap graphic representation by geo-enriching the data (assigning longitude and latitude to a location). Best practices indicate that it is better to create a blank model. This way, one first defines the dimensions and then uploads the data. However, the scope SAC does not allow Geo-Enrichment when creating a blank model. In SAP Analytics Cloud, one can create Custom Widgets (JavaScript code that can be imported into SAC). Using these custom widgets, one can create this graphical representation and do geo-enrichment. This was not in the scope of the internship so other functionalities were given priority.

Comparison Charts
This category contains graphical representations with which values in the data set can be compared.

Trend charts
This category includes graphics that can be used to identify trends in the data.

Distribution charts
This category includes graphical representations that can be used to show the distribution of data.

Correlation charts
This category contains graphical representations that allow the interrelationship of the data to be viewed.

Indicator charts
This category includes the bullet chart and the numeric point. In the bullet chart, the treshholds of the model can be displayed and with the numeric point, a value based on the filtering can be requested.

More
In this category you will find the graphics: Donut, Marimekko and Pie chart.

Application

User interface


In order to increase the ease of use and workflow of the application, a user can perform the following functionalities.

A user can hide or show fields for transferring data (for more information see chapter application -> planning functionality: transferring data) by using the hide/show button via the following path. Startpage -> button “hide/show”.

A user can apply filtering to the application (see chapter application -> filtering). To enhance the user experience, a user can reset the table's initial filters in the following way: Startpage -> button “RestoreFilters”

Filtering


A user can apply filtering to the table. This can be achieved in two ways. One can filter directly in the application in the following way

Startpage -> button “Filter” -> input the desired filtering -> button “Ok”.

A user can also cancel the filtering by clicking on the cancel button in the pop-up.

A second way that a user can apply filtering to the table is with the chatbot (see chapter chatbot -> filtering table).

Planningsfunctionality: transfer data


A user can also choose to transfer the data to another date. For example, he can choose to transfer all FTE values from 2019 to 2020. However, a user cannot plan in the past and therefore cannot plan the above use case. He can cancel the action at any time via the cancel button in the pop-ups.

A user can choose to transfer the complete dataset in the following way:

Homepage -> "Transfer All" button A user can also choose to transfer only the data for a specific year. This can be done in the following ways:

Start page -> "Date transfer" button -> in the pop-up, enter only the desired years -> "transfer on time" button.

Or

In the application, fill in the fields "from" and "to" and click on the button "transfer on time".

A user can also choose not to transfer the data of all cost centres for a specific year, but for 1 specific cost centre. He can do this in the same way, but fill in the input fields for company code and cost centre and click on the button "full input transfer".

Planningsfunctionality: transfer endbalance


Transfer all the endbalance values
A user can also choose to transfer the endbalance of each cost centre to the opening balance of the following months. He can do this in the following ways:

Startpage -> button "CarryAllEndBalance".

Or

By using the chatbot (see chapter chatbot -> transfer endbalance)

A user can also transfer a final balance for a specific cost centre in a specific company. He can do this in the following way:

Startpage -> button "CarrySpecificEndBalance" -> fill in the desired input in the pop-up -> OK

Planningsfunctionality: Planning FTE value
A user can also directly schedule FTE values in the table. He cannot schedule values in the past and not on the opening balance.

Chatbot

Introduction


In addition to performing planning functionalities directly in the story and application, a user can also do this with the help of the chatbot. Below, the different functionalities are explained and how a user can execute them. There is also a clear overview of the different commands. At the end it is explained how a user can customise the chatbot and control its own functionalities.

A chatbot must be trained, built and connected. Training the chatbot relates to learning intents that a user can perform. Building the chatbot relates to dealing with these intents and how the chatbot should deal with them. In the implementations, this is where the webhook is configured that establishes communication between both technologies. In addition, the chatbot is also connected which consists of a conversational AI web client connection.

The chatbot makes use of an algorithm called NLP (Natural Language Processing). This algorithm ensures that the chatbot assigns the intent that most closely matches the sentence. These intents contain entities which will become our usable data in SAP Analytics Cloud. This allows the chatbot to recognise similar commands and allows it to be flexible with errors, among other things.

Finally, a user can also monitor interactions with the chatbot.

Planning FTE Values

A user can schedule FTE values just like in the story and the application. The chatbot is trained to do this using the intent "fteinputtask". A user can perform this scheduling functionality with the chatbot using the command below:

Add 5 FTE for account Function for 202302 for company AT11 for cost center AT11_9302.

From the above command, the chatbot extracts the following entities:

FTEValue, Account, Date, CompanyCodeId, CostCenterNumber

Filtering table


A user can filter the table just like in the story and application. The chatbot is trained to do this using the intent "filtertable". A user can perform this scheduling function with the chatbot using the command below:

filter table on account OpeningBalance for company AT11 for costcenter AT11_9302 for 2019

From the above command, the chatbot extracts the following entities:

Account, CompanyCodeId, CostCenterNumber, Date.

Transfer endbalance


A user can transfer the final balance in the table to the next month. He can do this for one specific cost centre but also for all cost centres for a specific year.

The chatbot is trained to transfer all the end balances by means of the intent "transferallendbalance" and can be executed by a user with the command below: Transfer all the end balance values to the next month.
The chatbot recognises this sentence and this intent is passed to SAC. No entities need to be derived from this command.

The user can also use the chatbot to transfer the end balance for one specific cost centre at a certain time. The chatbot is trained for this using the intent "transfer specific balance". A user can perform this functionality by using the command below: transfer the endbalance for company AT11 for costcenter AT11_9302 from 2019
From the above command, the chatbot retrieves the following entities:

CompanyCodeId, CostCenterNumber, Date.

Transfer data


A user can use the chatbot to transfer data to another period. He can choose to do this for all costcenters or 1 specific costcenter, just like directly in the application. The chatbot is trained to do this using the intent "transfer data".

A user can execute the planning function to transfer all costcenters for a specific date by using the command below:

Trasfer FTE Values for from 2020 to 2021


From the above command, the chatbot extracts the following entities:

Date

A user can execute the scheduling function to transfer the values of 1 costcenter on a certain date by using the command below:

Trasfer 5 FTE for account New Function for company AT11 for cost center AT11_9302 from 2020 to 2021

From the above command, the chatbot extracts the following entities:

Date, CostCenterNumber, CompanyCodeId.

Graphical representations


Within SAP Analytics Cloud, data can be presented in many different ways using graphical representations. In order to make the right selection and choose the most suitable graphics, a communication standard called IBCS is applied. This standard determines which graphics are the most suitable to get the message across as efficiently as possible. The purpose is to find out the state of the company and how this can be done, and which graphics are chosen to do this, can be found below.

When a user wants to show the evolution of an account, a time series will be chosen as graphic. The chatbot is trained to do this using the intent "timeseries". A user can perform this functionality by using the following command:

Give me the evolution over time for account Netchange for company AT11 for date 2019 for costcenter AT11_9302

From the above command, the chatbot retrieves the following entities:

Account, CompanyCodeId, CostCenterNumber, Date

When a user wants to retrieve one particular value and display it on a graph, this is done using a numeric point. For this purpose the chatbot is trained using the intent "numericpoint". The user can perform this functionality by using the following command:

Amount of account EndBalance for 2019 for company AT11 for cost centre AT11_9302
From the above command, the chatbot retrieves the following entities:

Account, CompanyCodeId, CostCenterNumer, Date

When the user wants to see the relationship between the various accounts, a Waterfall Chart is chosen as the graphical representation. The chatbot is trained for this using the intent "givewaterfall". A user can invoke this functionality by using the command below: Show me the correlation of the accounts
The chatbot recognises this sentence and this intent is passed to SAC. No entities need to be derived from this command.

Resetting original filters


A user can choose to restore the original filters of the table not by using the button but by using the chatbot. The chatbot is trained to do this using the intent "restoreoriginalfilter". The user can invoke this functionality by using the command below: Restore the original Filter values
The chatbot recognises this sentence and this intent is passed to SAC. No entities should therefore be derived from this command.

Greeting the chatbot


In addition to performing the planning functions, filtering the table and retrieving graphics, a user can also greet the chatbot.
